1. Executive Summary 
Pact is a privacy-first, local-first group expense tracker and bill splitter built for tight-knit friend 
groups, roommates, and travel companions. Unlike incumbents such as Splitwise, Pact 
operates entirely on-device with zero mandatory account creation — making it structurally 
impossible for user financial data to be monetized or leaked. 
 
The product is designed to scale from a personal-use utility into a defensible consumer startup 
by embedding AI-powered intelligence, visual debt graph analysis, and a group financial health 
score — features no funded competitor can adopt without dismantling their core data 
monetization model. 
 
 
Strategic Insight 
Minimalist's acquisition by HUL in 2025 proved that radical transparency creates lasting consumer 
trust in otherwise opaque industries. Pact applies the same philosophy to personal finance 
between friends: full transparency, zero surveillance, community-first architecture. 
 
2. Problem Statement 
2.1 The User Problem 
Splitting expenses with friends is universally painful. Current tools either: 
• Require all participants to create accounts, creating friction and drop-off 
• Store sensitive financial data on remote servers, creating privacy risk 
• Lack intelligent suggestions — forcing manual categorization and split math 
• Provide no visual understanding of group financial dynamics 
• Offer no recurring expense tracking, leading to forgotten obligations

4. Feature Specification 
4.1 Core Features (MVP — v1.0) 
4.1.1 Group & Expense Management 
The foundational split-bill and expense tracking layer — feature-parity with Splitwise plus 
improvements. 
 
• Create named groups with optional emoji/avatar (no sign-up required) 
• Add members by name only — no email, phone, or account needed 
• Add expenses: amount, description, payer, split type, date, category 
• Split modes: Equal, Exact Amount, Percentage, Shares-based 
• Edit and delete expenses with full history log 
• Settle debts — mark payments, log UPI/cash/bank references 
• Multi-currency support with manual exchange rate input 
• Export group ledger as CSV or PDF 
 
4.1.2 Local-First Privacy Architecture 
Pact's defining architectural moat. No data leaves the device unless the user explicitly initiates 
an export. 
 
• All data stored in browser localStorage / IndexedDB 
• No user accounts, no sign-in, no email collection 
• Optional encrypted JSON export for backup (user-controlled) 
• Optional QR-code-based group sync for inviting friends (P2P, no server) 
• Clear data button with explicit user confirmation

Privacy Promise (Core Brand Commitment) 
Pact will never store user financial data on external servers without explicit opt-in. This is not a 
policy — it is a technical constraint baked into the architecture. There is no server to breach. 
 
4.1.3 Debt Graph Visualization 
The most visually differentiating feature of Pact. An interactive network graph showing who 
owes whom, with smart settlement path optimization. 
 
• Interactive force-directed graph of group members as nodes 
• Edges represent debts — thickness = magnitude, color = direction 
• Smart settlement optimizer: suggests minimum transactions to clear all debts 
• Example: A owes B ₹200, B owes C ₹200 → Pact suggests A pays C directly 
• Tap a node to see that member's full debt breakdown 
• Animate settlement when debts are marked as paid 
 
4.1.4 AI-Powered Smart Split Suggestions 
On-device AI (via Gemini Flash API) that reads context and suggests fair splits. 
 
• Analyze expense description to suggest category automatically 
• Suggest relevant group members to include based on past patterns 
• Detect unequal split scenarios (e.g., dietary restrictions, non-attendees) 
• Natural language expense entry: 'Dinner at Tatva, 6 people, ₹3200' → auto-fills form 
• AI chat: limited queries per day (privacy-safe, no conversation stored) 
 
4.1.5 Receipt Scanning 
Upload or capture a receipt photo — AI extracts line items, total, and suggests splits. 
 
• Camera capture or image upload 
• OCR + AI extraction of: merchant, total, line items, date 
• Line-item level splitting: assign different items to different people 
• Works offline for camera capture; AI processing uses Gemini Flash API (free tier) 
• Receipt image stored locally, never uploaded without user action 
 
4.1.6 Recurring Expense Tracking 
Track shared subscriptions, rent, utilities, and other recurring costs. 
 
• Create recurring expenses: weekly, monthly, custom interval 
• Auto-generate expense entry on due date with local notification

• Template system: save a split configuration and reuse it 
• Dashboard widget showing upcoming recurring obligations 
• History view: all past instances of a recurring expense 
 
4.1.7 Friend Group Financial Health Score 
A novel, proprietary metric unique to Pact. Computed entirely on-device from group expense 
patterns. 
 
• Score: 0–100, displayed as a ring gauge with trend indicator 
• Factors: settlement velocity, balance symmetry, expense frequency, reciprocity ratio 
• Breakdown: explains what is dragging the score down 
• Historical trend: 30/60/90 day score graph 
• Insights: 'Your group settles within 3 days on average — top 15%' 
 
 
Why This Matters 
No expense tracker has ever gamified financial health at a group level. This feature creates 
emotional stickiness — users will check their score the way they check their credit score. It is also 
uniquely defensible because the scoring model is proprietary. 
 
4.2 Secondary Features

• Personal expense tracker (non-group, private journal of spending) 
• Expense categories with custom emoji + color 
• Spending analytics: charts by category, member, time 
• Activity feed: chronological log of all group actions 
• Search and filter expenses by date, amount, category, member

6. UX & Design System 
6.1 Design Principles 
• Clarity over cleverness — every screen has one primary action 
• Data density is a feature, not a bug — power users should see more 
• Motion with purpose — animations communicate state, not decoration 
• Privacy signals everywhere — reinforce the local-first message in UI copy

9. Appendix 
9.1 Glossary 
• Local-first: Architecture where data is stored on the user's device as the primary source of 
truth 
• Debt graph: Network visualization of financial obligations between group members 
• Settlement optimizer: Algorithm that reduces N debts to minimum required transactions 
• Health score: Proprietary 0–100 metric measuring a group's financial relationship quality 
• PWA: Progressive Web App — installable web app with offline support
